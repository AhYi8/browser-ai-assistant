## v3.7.0 - 2026-07-15

本版本新增扩展更新提醒能力。浏览器启动或扩展完成安装、更新后会检查 GitHub 最新正式 Release；当本地版本落后时，侧边栏顶部显示绿色更新图标，点击即可在新标签页打开对应 Release 页面。

### 发布范围

本次版本发布覆盖 `v3.6.1` 之后的正式版本更新提醒提交：

- `a704fcb`：增加 GitHub 正式 Release 检测、顶部更新入口、版本与 URL 安全校验、并发去重、请求超时和回归测试。

### 新增

- 新增 GitHub 最新正式 Release 自动检测，在扩展安装、更新和浏览器启动时执行一次。
- 当本地版本低于远程正式版本时，在 `app-header-actions` 最左侧显示更新图标。
- 更新图标使用绿色圆圈和向上箭头，按钮尺寸、边框和交互延续现有头部操作样式。
- 点击更新图标后通过 `chrome.tabs.create` 在新标签页打开对应 Release 页面。
- 版本标签兼容 `v3.7.0` 与 `3.7.0` 两种三段数字格式。

### 可靠性与安全

- 只检查 GitHub 最新正式 Release，预发布版本和草稿不会触发更新提示。
- 严格校验版本标签、仓库地址和 HTTPS 协议，拒绝空白标签、非法后缀、查询参数、锚点及越界链接。
- 使用 GitHub `/releases/latest` 普通页面重定向获取版本，不依赖 GitHub API Token，也不消耗匿名 REST API 配额。
- GitHub 检测请求设置 10 秒超时，网络失败时保留最近一次成功缓存，不阻断 Side Panel 主流程。
- 安装与浏览器启动事件紧邻触发时复用同一个 in-flight 请求，避免重复访问和缓存竞态。
- 同时兜底 `chrome.tabs.create` 的同步抛错与 Promise 拒绝，避免产生未处理异常。

## v3.6.1 - 2026-07-15

本版本为历史会话列表新增批量管理能力，支持按文件夹选择会话并批量归档或删除。同时优化批量操作区域的布局与交互状态，让桌面侧栏和窄屏历史弹窗都能保持紧凑、清晰和稳定。

### 发布范围

本次版本发布覆盖 `v3.6.0` 之后的历史会话批量操作提交：

- `a5db2b1`：增加历史会话批量归档、批量删除、文件夹级选择和紧凑批量控制区。

### 新增

- 历史会话列表新增“批量操作”模式。
- 未归档分区支持批量归档和批量删除，已归档分区支持批量删除。
- 每个文件夹提供独立的“全选 / 取消全选”入口，全选仅作用于当前文件夹。

- 桌面历史侧栏与窄屏历史弹窗分别维护批量选择状态，互不影响。

### 优化

- “批量操作”入口固定在历史会话头部操作区，进入批量模式后使用高亮状态提示。

- 批量模式下禁用会话打开、重命名、拖拽和单条操作，避免交互冲突。

### 可靠性

- 批量归档和删除使用单次事务完成，避免出现部分成功、部分失败的状态。
- 批量归档只更新归档状态和更新时间，避免覆盖会话中的其他最新内容。
- 删除运行中的会话前先中止对应任务，并在删除成功后统一清理相关运行状态。
- 操作失败时保留原会话和当前选择，并显示明确的中文错误提示。
- 批量操作完成后自动退出批量模式，并重新解析当前活动会话。

### 工程

- Vitest 恢复使用默认排除规则，避免误收集 `relay/node_modules` 中的第三方依赖测试。
- 更新最终回答附件与上下文压缩相关历史断言，使测试契约与当前附件上下文边界一致。

## v3.5.0 - 2026-07-04

本版本新增“运行中跟进对话”能力：AI 回复或工具调用尚未结束时，用户也可以继续输入补充内容，并选择排队到下一轮发送，或作为引导影响当前任务的后续模型决策。

### 发布范围

本次版本发布覆盖当前 `master` 相对 `origin/master` 的本地变更，重点包含运行中跟进对话、引导机制和排队队列 UI。

### 新增

- 新增聊天偏好“跟进行为”，支持排队与引导两种模式。
- AI 运行中仍可编辑并提交聊天输入框内容。
- 运行中输入框有草稿时主按钮显示“发送”，无草稿时显示“终止”。
- 输入框上方新增排队队列区域，支持折叠预览、展开查看、删除单条和单条改为引导。
- 引导消息会立即以用户消息气泡展示。
- 引导被实际消费后，会显示“已引导对话”过程提示。
- `Ctrl+Shift+Enter` 在运行中固定执行当前跟进行为的相反操作。

### 优化

- 排队队列按会话隔离，后台会话完成后会继续写回原会话。
- 用户主动终止当前任务后，队列仍会按顺序继续消费。
- 引导跟进会保留文本、图片附件和 Prompt 调用快照。
- 已切换为引导的跟进项不再显示在排队队列中。
- 排队跟进自动消费失败时会保留原队列项，避免用户输入丢失。
- 旧版 `Ctrl+Shift+Enter` 发送快捷键配置会自动迁移为普通 Enter 发送。

### 修复

- 修复非运行中 `Ctrl+Shift+Enter` 可能绕过发送快捷键偏好的问题。
- 修复后台普通会话自动消费排队跟进时可能受当前隐私模式影响的问题。
- 修复带图片或 Prompt 快照的引导无法完整进入工具循环的问题。
- 修复排队队列自动消费前置校验失败时可能丢失队列项的问题。

## v3.3.0 - 2026-07-02

本版本新增 MCP 远程工具集成能力，第一版以 MVP 策略接入 HTTP/Streamable HTTP MCP Server 的 Tools 能力，不支持 stdio、resources、prompts、OAuth 或自定义 header。

### 新增功能

- 新增 MCP 工具设置页，可新增、编辑、删除、启用/停用 HTTP MCP Server。
- 支持可选 Bearer Token，本地敏感存储，不进入同步快照。
- 支持自动发现 MCP 工具，并接入现有聊天工具调用体系。
- 新增 MCP 远程工具运行类型 `mcp_remote`，独立于浏览器自动化权限。
- 新增 MCP 工具在聊天偏好和当前聊天工具菜单中的启用/禁用能力。
- 新增 MCP Server 保存后自动刷新工具列表，并默认启用新发现的 MCP 工具。
- MCP 工具在工具列表中以 `MCP名称.tool_name` 形式展示，保留内部工具 ID 和模型侧函数名稳定不变。

### 安全与边界

- MCP 工具执行前在 background 二次校验 Server、工具名、启用状态和发现缓存。
- MCP 工具不受普通模式、受控增强、完全访问等浏览器自动化模式影响。
- MCP HTTP 请求增加默认超时，避免远端无响应导致聊天流程挂起。
- MCP Server 删除增加二次确认。
- 修复 MCP 工具 ID 解析对点号 `serverId` 的歧义。
- 修复 MCP 模型侧工具名冲突时可能误执行其他远程工具的问题。
- background 刷新 MCP 工具列表时只接收 `serverId`，不信任 UI 传入的完整 Server 对象。

### 体验优化

- MCP 设置页工具列表默认收起，通过“工具列表”按钮展开。
- 工具列表标题和描述分两行展示，长文本截断并支持悬浮查看完整内容。
- MCP 工具调用过程复用内置工具调用样式，并受现有聊天偏好控制。
- 移除 MCP 参数摘要展示逻辑，完整参数仍通过调用详情查看。

- 通过 `npm run typecheck`
- 通过 `npm run build:extension`
- 通过 `npm run test`
- 通过 `npm run check`
- 已生成本地扩展打包产物

## v3.0.0 - 2026-06-26

本版本聚焦浏览器自动化与调试分析能力升级，将 Network 分析从 DevTools 页面迁移为基于 `chrome.debugger` 的后台采集，并补全 `network.*`、`js.*`、`sourcemap.*`、`runtime.*`、`replay.*`、`full_access.*` 以及页面观测、元素检查、截图、控制台采集等通用浏览器自动化工具。用户无需手动打开 DevTools，即可在开启浏览器控制后进行请求分析、源码检索、Source Map 映射、运行时只读分析和受控请求重放。
