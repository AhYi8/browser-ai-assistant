# v3.6.1 发布实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将历史会话批量操作作为 `v3.6.1` 发布，并上传可安装的 Chrome 扩展压缩包。

**Architecture:** 发布提交只更新版本元数据、变更日志、发布约束和本计划，不修改业务逻辑。发布资产由现有构建与打包脚本生成，验证通过后再创建 tag 和 GitHub Release，避免产生半发布状态。

**Tech Stack:** npm、Vite、Vitest、TypeScript、PowerShell、Git、GitHub CLI

---

### Task 1: 同步版本与发布说明

**Files:**
- Modify: `package.json`
- Modify: `package-lock.json`
- Modify: `public/manifest.json`
- Modify: `CHANGELOG.md`
- Modify: `AGENTS.md`
- Create ignored artifact: `artifacts/release-v3.6.1-notes.md`

- [ ] 使用 `npm version 3.6.1 --no-git-tag-version` 更新 package 与 lockfile 根版本。
- [ ] 将 `public/manifest.json` 版本更新为 `3.6.1`。
- [ ] 在 `CHANGELOG.md` 顶部记录 `v3.6.1` 的发布范围、新增、优化、可靠性与验证项。
- [ ] 在 `AGENTS.md` 记录 `v3.6.1` 发布范围和资产要求。
- [ ] 使用 UTF-8 编码写入最终 GitHub Release description。

### Task 2: 执行发布检查

**Files:**
- Verify: `package.json`
- Verify: `package-lock.json`
- Verify: `public/manifest.json`
- Verify: `artifacts/chrome-extension`

- [ ] 运行 `npm run check`，要求类型检查、构建、测试和本地打包全部通过。
- [ ] 检查三个版本文件与 `artifacts/chrome-extension/build-info.json` 均为 `3.6.1`。
- [ ] 检查打包目录不包含测试文件，manifest 与 HTML 引用资源完整。

### Task 3: 生成并校验发布资产

**Files:**
- Create ignored artifact: `artifacts/browser-ai-assistant-v3.6.1.zip`

- [ ] 从 `artifacts/chrome-extension` 创建 `browser-ai-assistant-v3.6.1.zip`。
- [ ] 检查 ZIP 非空并记录 SHA-256。
- [ ] 检查 ZIP 内根目录直接包含 `manifest.json`、`index.html`、`background/index.js` 和 `content/index.js`。

### Task 4: 提交、推送并发布

**Files:**
- Commit all tracked release changes.

- [ ] 执行 `git add .` 并提交 `chore(release): 发布 3.6.1`。
- [ ] 推送 `master`，确认本地 HEAD 与 `origin/master` 一致。
- [ ] 创建并推送注解 tag `v3.6.1`。
- [ ] 使用 `gh release create v3.6.1 --title v3.6.1 --notes-file artifacts/release-v3.6.1-notes.md` 创建 Release 并上传 ZIP。
- [ ] 读取远端 Release，核对 tag、标题、正文、资产名称、大小和摘要。
