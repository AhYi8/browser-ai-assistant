# AI 对话标题生成设计

## 目标

在渠道管理中增加“AI 标题生成模型”配置。用户未配置该模型时，保持当前会话标题逻辑：新会话使用首条用户消息前若干字符作为标题，不额外请求 AI。用户配置标题模型后，在首轮对话完成后使用该模型非流式生成标题，并更新当前会话标题。

## 范围

- 增加全局唯一的标题生成模型配置入口。
- 标题生成请求始终使用非流式。
- 标题模型使用严格 JSON 响应，应用只接受 `title` 字段。
- 标题生成失败、返回非 JSON、返回空标题或模型不可用时，保留当前默认标题。
- 不改变主聊天模型选择、流式回复、历史会话手动重命名和文件夹逻辑。

## 架构

已有 `ProviderModel.isTitleModel` 字段可表达标题模型。设置页通过一个下拉框选择标题模型，并确保同一时间最多只有一个模型的 `isTitleModel` 为 `true`。发送聊天消息时，首条用户消息先按当前逻辑创建默认标题；主回复完成后，如果存在启用的标题模型及其启用渠道，则构造标题专用 Prompt，发送非流式请求，并用 JSON 中的 `title` 更新会话。

## Prompt 与响应

标题请求包含系统消息和用户消息。系统消息要求模型只返回 JSON 对象，不返回 Markdown、解释或额外文本。响应格式固定为：

```json
{"title":"标题"}
```

解析逻辑只读取 JSON 对象中的 `title` 字符串；会去除首尾空白，并限制最大长度。任何非 JSON 文本、缺失字段、空字符串或异常都视为生成失败。

## 验收标准

- 渠道管理中可以选择或取消“AI 标题生成模型”。
- 未配置标题模型时，发送首条消息后标题仍是当前默认逻辑。
- 配置标题模型时，主 AI 回复完成后额外发起一次非流式标题请求。
- 标题模型返回 `{"title":"页面摘要讨论"}` 时，会话标题更新为 `页面摘要讨论`。
- 标题模型返回解释文本或无效 JSON 时，会话标题保持默认标题。
- 标题生成失败不影响聊天消息保存和主回复展示。

## 风险

- 现有部分源码中文字符串显示为乱码，本任务只修改必要位置，不顺手修复历史编码问题。
- 标题生成是额外模型调用，会增加一次请求成本；仅在用户显式配置标题模型后触发。
