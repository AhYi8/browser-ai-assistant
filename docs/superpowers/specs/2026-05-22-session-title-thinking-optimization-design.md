# 会话标题与思考状态优化设计

## 目标

优化历史会话标题展示、AI 标题生成时机和思考过程文案，让首轮会话启动后更快出现标题生成反馈，并减少历史列表冗余文本。

## 范围

- 历史会话列表不再在标题前显示 `会话：`。
- 历史会话标题按钮悬浮时显示完整标题。
- 配置 AI 标题生成模型后，首轮用户消息保存后立即并行生成标题。
- 标题生成期间会话标题位置显示等待态。
- 流式思考过程中显示 `思考中`，完成后显示 `思考过程`。

## 方案

采用 `ChatSession.titleGenerating?: boolean` 表示标题等待态。首轮发送消息时，如果存在启用的标题模型和启用渠道，则保存会话时写入 `titleGenerating: true`，同时启动非流式标题请求；主聊天请求继续照常发送。标题生成成功后更新标题并清除等待态；失败时保留默认标题并清除等待态。用户在标题生成期间手动重命名时，标题生成结果不得覆盖用户手动标题。

历史会话 UI 直接展示 `session.title`，通过原生 `title` 属性提供悬浮完整标题。`MessageList` 根据 `message.streaming` 切换 summary 文案：流式思考中为 `思考中`，结束后为 `思考过程`。

## 验收标准

- 历史列表中不再出现 `会话：资料会话`，只显示 `资料会话`。
- 长标题元素拥有完整标题提示。
- 配置标题模型后，发送首条消息会立即出现 `生成标题中...`，且标题请求与主聊天请求并行。
- 标题请求成功后替换标题；失败后保留默认标题并取消等待态。
- 流式思考中的 summary 是 `思考中`，完成后是 `思考过程`。
