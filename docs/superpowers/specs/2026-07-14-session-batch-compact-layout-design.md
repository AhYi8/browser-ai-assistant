# 历史会话批量操作紧凑布局设计

## 背景

历史会话批量归档与删除已实现，但进入批量模式后，头部的新建入口被移除，批量控制区同时使用分区、选中数量和操作按钮三行布局，占用了过多会话列表空间。

本次只调整批量模式的头部按钮状态与控制区密度，不改变批量归档、批量删除、分区校验、文件夹级全选、事务持久化和运行状态清理逻辑。

## 交互设计

历史会话标题区始终显示以下三个按钮，顺序保持不变：

1. 批量操作
2. 新建文件夹
3. 新建

进入批量模式后：

- “批量操作”按钮保持可见，并通过 `aria-pressed="true"`、主色边框和浅色背景表达激活状态。
- “新建文件夹”和“新建”按钮保持可见，但设置为禁用并使用现有禁用态样式置灰。
- 批量操作正在提交时，“批量操作”按钮也保持禁用，避免提交过程中退出模式。
- 退出批量模式后，三个按钮恢复普通状态。

## 紧凑控制区

采用方案 C，将批量控制区压缩为单行网格：

- 左侧为“未归档 / 已归档”分段控件。
- 未归档分区右侧显示“归档 N”和“删除 N”。
- 已归档分区右侧只显示“删除 N”。
- `N` 为当前分区内已选会话数量；数量为零时按钮保留并禁用，避免操作区宽高跳动。
- 原本可见的“已选 N 项”不再单独占行，但保留视觉隐藏的 `aria-live` 状态，确保辅助技术仍能获知选择变化。

控制区继续使用稳定网格，不使用绝对定位。分段控件占据剩余空间，操作按钮按内容获得稳定宽度；文字不得被截断或覆盖。桌面侧栏与窄屏历史弹窗共用同一组件和布局规则。

## 既有行为保持

- “全选/取消全选”仍位于各文件夹标题中，只作用于当前文件夹，不能跨文件夹选择。
- 切换“未归档 / 已归档”仍清空当前选择。
- 已归档分区不显示归档操作。
- 会话行在批量模式下仍禁用打开、重命名、拖拽和单条操作。
- 确认弹窗、成功退出、失败保留选择等行为不变。

## 验证

组件测试需要覆盖：

- 批量模式下三个头部按钮仍存在。
- “批量操作”处于按下状态，另外两个按钮禁用。
- 未归档分区显示带数量的归档和删除按钮。
- 已归档分区只显示带数量的删除按钮。
- 选中数量变化时按钮文案同步更新，零选择时按钮禁用。
- 文件夹级全选、分区切换、确认取消和确认执行的既有测试继续通过。

完成后运行批量功能相关单元测试、类型检查和扩展构建，并在桌面侧栏及窄屏历史弹窗中检查无溢出、无重叠和无异常换行。
