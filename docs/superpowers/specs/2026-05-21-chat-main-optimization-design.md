# 聊天主界面优化设计

## 背景

聊天主界面 MVP 已完成基础落地，包括宽窄响应式布局、历史会话、归档、聊天气泡、上下文提取模式和真实非流式请求。本轮优化聚焦在历史侧栏可用性、输入区控件形态、上下文查看方式和整体滚动体验，目标是在不重做架构的前提下提升插件日常使用效率。

## 目标

- 左侧历史对话模块宽度增加约 20%，从当前 `220px` 调整到约 `264px`。
- 会话项操作从外露小按钮改为菜单按钮，菜单内提供“重命名”“归档”“删除”。
- 支持新建文件夹，并支持把历史会话拖拽移动到指定文件夹。
- AI 消息在存在思考内容时显示“思考过程”折叠模块，默认折叠。
- 输入区的“流式响应”和“提取文本 / 提取所有”改为 switch 外观，不再显示原生复选框。
- Side Panel 不出现全局滚动条，仅历史普通分组、聊天消息展示区和弹窗内容按需内部滚动。
- “查看上下文”从 `details` 折叠改为按钮打开弹窗，弹窗内部展示上下文。

## 非目标

- 不实现文件夹拖拽排序。
- 不实现会话在文件夹内排序。
- 不实现批量移动、批量删除或历史搜索。
- 不实现归档会话解除归档。
- 不改变聊天请求和上下文提取的后端契约。
- 不重做设置页或处理设置页既有硬编码颜色技术债。

## 交互设计

### 历史侧栏

宽面板时历史侧栏仍显示在左侧，宽度提升到约 `264px`，右侧聊天区域保持 `minmax(0, 1fr)`，避免长模型名或长消息撑出横向滚动。

历史侧栏顶部提供：

- 标题“历史对话”。
- “新建文件夹”按钮。
- 现有“新建对话”入口继续保留。

普通文件夹区域独立滚动，已归档区域继续放在底部并默认折叠，不随普通会话列表滚走。

### 会话菜单

每条会话标题和菜单按钮在同一行：

- 左侧显示会话标题，长标题省略。
- 右侧显示 `⋯` 菜单按钮。
- 点击菜单按钮弹出子菜单。

子菜单包含：

- `重命名`
- `归档`，仅未归档会话显示。
- `删除`

删除仍沿用两段式确认：首次点击“删除”后，该菜单项文案变为“确认删除”；再次点击才真正删除。关闭菜单或切换会话后应退出待确认状态。

### 重命名

重命名采用原地输入框：

- 点击“重命名”后，会话标题位置切换为输入框。
- `Enter` 保存。
- `Escape` 取消。
- 输入框失焦时保存非空标题；空标题不保存。

这种方式比弹窗更轻，适合频繁整理历史对话。

### 新建文件夹

点击“新建文件夹”后：

- 创建一个默认名为“新文件夹”的文件夹。
- 新文件夹出现在普通文件夹列表底部。
- 创建后立即进入文件夹名原地编辑状态。

文件夹名称空值不保存；如果用户取消或输入空值，保留默认名“新文件夹”。

### 拖拽移动会话

会话项支持 HTML5 拖拽：

- 拖拽开始时记录 `sessionId`。
- 文件夹标题或文件夹区域作为可投放目标。
- 拖拽悬停时显示轻量投放态。
- 投放到指定文件夹后，更新该会话的 `folderId`。
- 投放到默认文件夹后，清空该会话的 `folderId`。

归档会话不参与拖拽移动。文件夹自身不支持拖拽排序。

## 聊天区设计

### 消息展示区

聊天主面板采用固定高度布局：

- 顶部模型行固定。
- 中间消息展示区 `flex: 1; min-height: 0; overflow: auto`。
- 底部输入区固定。
- 外层 `app-shell` 和聊天主布局不产生全局滚动。

这样页面不出现全局滚动条，长聊天记录只在消息展示框内部滚动。

### 思考过程

AI 消息存在 `thinking` 时，在 AI 回复气泡上方显示“思考过程”折叠模块：

- 默认折叠。
- 使用 `details` / `summary` 或等价可访问结构。
- 仅 `thinking` 存在时显示；普通 AI 回复不显示该模块。

## 输入区设计

### Switch 开关

`流式响应` 和 `提取文本 / 提取所有` 改为 switch 外观：

- 使用原生 `input type="checkbox"` 保持键盘和读屏可访问性。
- 原生输入视觉隐藏，但仍可聚焦。
- 自定义轨道和圆点展示开关状态。
- 关闭提取模式时显示“提取文本”。
- 开启提取模式时显示“提取所有”。

### 查看上下文弹窗

输入区中的“查看上下文”改为按钮：

- 点击后打开上下文弹窗。
- 弹窗标题为“当前页上下文”。
- 弹窗内展示当前 `pageContext.text`，无内容时显示“暂无上下文”。
- 上下文内容区域内部滚动，不改变聊天主布局高度。
- 弹窗提供关闭按钮，支持 `Esc` 关闭。

## 数据与状态

需要在现有 store 上补充或扩展以下动作：

- `createChatFolder()`：创建文件夹。
- `renameChatFolder(folderId, name)`：重命名文件夹。
- `moveChatSessionToFolder(sessionId, folderId | undefined)`：移动会话到文件夹或默认文件夹。
- `renameChatSession(sessionId, title)`：供菜单原地重命名调用；如已存在则复用。

存储层已有 `ChatFolder` 和 `ChatSession.folderId`，本轮优先复用现有数据结构，不新增数据库表。

## 错误处理

- 新建文件夹或移动会话持久化失败时，显示现有全局失败提示，保留界面原状态或重新加载存储数据。
- 重命名为空时不保存，避免产生不可识别的历史项。
- 拖拽过程中如果目标文件夹不存在，不执行移动。

## 测试验收

组件测试覆盖：

- 历史侧栏宽度类或布局样式变更。
- 会话菜单打开后显示“重命名 / 归档 / 删除”。
- 删除菜单项需要“删除 -> 确认删除”两步。
- 重命名会话可原地编辑并保存。
- 新建文件夹后显示文件夹，并可重命名。
- 拖拽会话到文件夹后，会话移动到目标文件夹。
- `流式响应` 和 `提取文本 / 提取所有` 使用 switch 结构与文案。
- 点击“查看上下文”打开弹窗并显示上下文内容。
- 外层布局不产生全局滚动，消息区为内部滚动容器。
- AI thinking 存在时默认折叠显示“思考过程”。

## 风险

- HTML5 拖拽在触控设备上体验有限；Side Panel 主要面向鼠标操作，本轮可接受，后续如需触控友好可补移动按钮。
- 历史侧栏变宽会压缩聊天区；已通过 `minmax(0, 1fr)` 和窄屏隐藏侧栏降低影响。
- 菜单和重命名状态增加了局部 UI 状态复杂度，需要确保切换会话、关闭菜单和删除确认之间不会互相污染。
