# 默认对话模型设计

## 目标

在渠道管理中增加“默认对话模型”配置，并放在“AI 标题生成模型”上方。新建对话时默认切换到该模型，当前对话仍允许通过聊天区“当前模型”临时切换。

## 方案

默认对话模型是全局偏好设置，使用现有 `appSettings` 表保存 `defaultChatModelId`。这样不需要修改模型数据结构，也避免把“模型是否默认”混入渠道模型本身。

## 行为

- 渠道管理展示“默认对话模型”下拉框，选项来自当前所有可用渠道和模型。
- 用户选择模型后保存到 `appSettings`。
- 用户清空配置后，新建对话回退到第一个可用模型。
- 配置的默认模型被删除、所属渠道被删除或不可用时，新建对话回退到第一个可用模型。
- 新建对话只设置当前选中模型，不给空会话增加消息字段。

## 验证

- 单元测试覆盖默认模型保存、加载、删除回退和新建对话选择。
- 组件测试覆盖渠道管理中默认模型下拉框的位置与交互。
- 运行相关 Vitest、类型检查和构建。
